# STM32F4_MFRC522

An STM32-based driver and example application for the **MFRC522** RFID reader/writer module. The project targets the **STM32F446RE** microcontroller on the **NUCLEO-F446RE** development board and implements a complete ISO/IEC 14443-A communication stack supporting MIFARE Classic 1K tags — including anticollision/select, authentication, block read, and block write operations.

---

## Target Hardware

| Item | Details |
|------|---------|
| MCU | STM32F446RE (ARM Cortex-M4, 180 MHz) |
| Board | NUCLEO-F446RE |
| RFID Module | MFRC522 (NXP) — 13.56 MHz contactless reader/writer |
| Tag support | ISO/IEC 14443-A, MIFARE Classic 1K |
| Toolchain | arm-none-eabi-gcc, CMake (STM32CubeMX generated) |

## Pin Assignments

### SPI2 — MFRC522 Interface

The MFRC522 is connected over **SPI2** using remapped alternate-function pins. Software-controlled chip select (active low) is used instead of the hardware NSS line.

| Signal | STM32 Pin | Direction | Notes |
|--------|-----------|-----------|-------|
| SCK | **PB10** | Output | SPI2 clock, AF5 |
| MOSI | **PC1** | Output | SPI2 data out, AF5 |
| MISO | **PC2** | Input | SPI2 data in, AF5 |
| CS (NSS) | **PC3** | Output | Software chip select, active low |

SPI is configured as full-duplex master at approximately **5.25 MHz** (APB1 ÷ 8).

### MFRC522 Control Pins

| Signal | STM32 Pin | Direction | Notes |
|--------|-----------|-----------|-------|
| RST | **PC0** | Output | Hard reset — driven low to reset the MFRC522, then released high |
| IRQ / EXTI | **PA1** | Input | Interrupt output from the MFRC522; falling edge, internal pull-up; routed to **EXTI1** |

The IRQ line is used in the interrupt-driven card detection example (`Example_ReadUID_IRQ`). The MFRC522 is configured to assert IRQ low when a card response is detected in its FIFO.

### GPIO — User I/O

| Signal | STM32 Pin | Direction | Notes |
|--------|-----------|-----------|-------|
| LED (LD2) | **PA5** | Output | Built-in green LED on the Nucleo board; toggled to indicate scan activity or auth result |
| Buzzer | **PA0** | Output | External passive/active buzzer; driven high on a successful IRQ card detect, then low after a short delay |
| User Button (B1) | **PC13** | Input | Built-in blue push-button; falling-edge EXTI13 interrupt (not used in current examples) |

### USART2 — Debug Output

Serial output is routed through the Nucleo's on-board ST-Link virtual COM port.

| Signal | STM32 Pin | Notes |
|--------|-----------|-------|
| TX | **PA2** | 115200 baud, 8N1 |
| RX | **PA3** | |

`printf` is retargeted to USART2 via the syscalls stub, so all `DEBUG_LOG(...)` output appears on the virtual COM port.

### SWD — Debug/Programming

| Signal | STM32 Pin |
|--------|-----------|
| SWDIO / TMS | PA13 |
| SWCLK / TCK | PA14 |
| SWO | PB3 |

## Wiring — MFRC522 Module to Nucleo-F446RE

```
MFRC522 Module          NUCLEO-F446RE
─────────────           ─────────────
3.3V         ──────────  3.3V
GND          ──────────  GND
RST          ──────────  PC0
SDA (CS)     ──────────  PC3
SCK          ──────────  PB10
MOSI         ──────────  PC1
MISO         ──────────  PC2
IRQ          ──────────  PA1
```

> **Note:** The MFRC522 module operates at **3.3 V logic**. Do not connect it to 5 V I/O lines. The STM32F446RE GPIO pins are 3.3 V, so no level shifting is required.

## Driver API Overview

The driver is initialised with a handle that binds it to a specific SPI peripheral and CS/RST pins, making it portable across different hardware configurations:

```c
MFRC522_Init(&mfrc522, &hspi2,
             SPI2_CS_GPIO_Port, SPI2_CS_Pin,
             MFRC522_RST_GPIO_Port, MFRC522_RST_Pin);
```

Key operations:

| Function | Description |
|----------|-------------|
| `MFRC522_IsCardPresent` | Sends REQA; returns `MFRC522_OK` if a card responds |
| `MFRC522_AntiCollision` | Runs anticollision loop and full select (SAK verified) |
| `MFRC522_ReadUID` | Combines `IsCardPresent` + `AntiCollision`; returns 4-byte UID |
| `MFRC522_MifareAuth` | Authenticates a sector using Key A or Key B |
| `MFRC522_MifareRead` | Reads a 16-byte block (requires prior auth); validates CRC |
| `MFRC522_MifareWrite` | Writes a 16-byte block (requires prior auth) |
| `MFRC522_DisableCrypto1` | Clears the MFCrypto1On bit — must be called after each auth session |
| `MFRC522_Enable_Antenna` / `MFRC522_Disable_Antenna` | RF field control |

## Examples

Select the example to run by passing an `Example_t` value to `Examples_Run`:

| Example | Description |
|---------|-------------|
| `Example_ReadUID` | Polling loop — reads and prints the UID of any presented card |
| `Example_ReadUID_IRQ` | IRQ-driven — MFRC522 asserts PA1 when a card is detected; buzzer beeps on success |
| `Example_ReadWrite` | Authenticates sector 1 (block 4) with Key A, reads the block, increments a counter value, and writes it back |

## Building

The project uses CMake with a Ninja generator. A preset for the Debug configuration is provided:

```bash
cmake --preset Debug
cmake --build --preset Debug
```

Flash with OpenOCD, ST-Link Utility, or directly from VS Code using the Cortex-Debug extension.
